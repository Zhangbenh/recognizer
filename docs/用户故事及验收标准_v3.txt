# 《用户故事与验收标准（工程规范版 V3）》
## 便携式植物识别与区域采样系统（云端优先，本地回退）

# v1.1 补丁回填说明

自 v1.1 起，本文件新增并回填以下需求口径：

- 默认中文显示
- 地图与区域图片化选择
- 云端优先识别与本地回退
- 云端新增植物允许通过稳定命名空间 `plant_key` 直接写入统计
- 新增地图级汇总统计查看

若本文件旧版条目与上述回填内容冲突，以本节及后续修订章节为准。

# 一、用户故事分层

- Epic（用户目标）

# Epic 1：核心识别体验

---

## Feature 1.1 实时取景系统

### US-1.1.1 实时取景显示

**User Story**

- As 用户  
- I want 实时看到摄像头画面  
- So that 我能够准确对准目标植物  

**验收标准**

- Given 设备开机  
  When 系统初始化完成  
  Then 在2秒内进入取景界面  

- Given 取景界面运行中  
  When 系统稳定运行  
  Then 帧率应 ≥15 FPS  

- Given 取景界面  
  When 用户观察屏幕  
  Then 屏幕中央显示准星  

- Given 连续运行10分钟  
  When 用户持续使用  
  Then 系统无卡死或崩溃  

- Given 摄像头异常  
  When 无法获取图像  
  Then 显示错误提示  

---

## Feature 1.2 图像抓取

### US-1.2.1 按键拍照

**User Story**

- As 用户  
- I want 快速拍摄当前画面  
- So that 捕捉清晰的植物图像  

**验收标准**

- Given 用户按下拍照键  
  When 按键触发  
  Then 系统响应时间 ≤200ms  

- Given 拍照触发  
  When 图像冻结  
  Then 冻结误差 ≤1帧  

- Given 拍照成功  
  When 图像生成  
  Then 输出有效图像数据  

- Given 用户连续按键  
  When 触发频率过高  
  Then 系统不产生重复拍照  

---

## Feature 1.3 云端优先识别

### US-1.3.1 云端优先识别与本地回退

**User Story**

- As 用户  
- I want 优先使用云端识别，并在失败时自动回退本地模型  
- So that 提升识别成功率，同时保留离线可用性  

**验收标准**

- Given 网络可用  
  When 用户发起识别  
  Then 系统优先请求云端识别  

- Given 云端请求超时或失败  
  When 识别流程继续  
  Then 系统自动回退本地模型  

- Given 图像输入  
  When 走云端成功路径  
  Then 云端请求时间 ≤3秒  

- Given 图像输入  
  When 走本地回退路径  
  Then 本地模型推理预算 ≤1.5秒  

- Given 图像输入  
  When 整体识别流程完成  
  Then `INFERENCING` 总体预算 ≤4.5秒  

- Given 无网络连接  
  When 用户发起识别  
  Then 系统仍可通过本地模型完成识别  

- Given 推理执行中  
  When 系统处理  
  Then 显示“AI识别中”提示  

- Given 推理成功  
  When 识别完成  
  Then 返回分类结果  

- Given 推理失败  
  When 云端与本地都失败  
  Then 显示失败提示  

---

## Feature 1.4 结果展示

### US-1.4.1 识别结果显示

**User Story**

- As 用户  
- I want 清晰看到识别结果  
- So that 理解识别结果的可信度  

**验收标准**

- Given 识别成功  
  When 展示结果  
  Then 显示植物名称  

- Given 识别成功  
  When 展示结果  
  Then 显示置信度  

- Given 显示结果  
  When UI渲染  
  Then 结果位于屏幕底部1/3区域  

- Given 识别失败  
  When 无有效结果  
  Then 显示“未识别”提示  

---

## Feature 1.5 流程闭环

### US-1.5.1 完整识别流程

**User Story**

- As 用户  
- I want 完成一次完整识别流程  
- So that 可以连续进行识别  

**验收标准**

- Given 用户进入取景  
  When 执行完整操作  
  Then 流程为：取景→拍照→推理→显示  

- Given 显示完成  
  When 用户未操作  
  Then 5秒内自动返回取景  

- Given 显示结果  
  When 用户操作  
  Then 支持手动返回取景  

- Given 任意流程阶段  
  When 系统运行  
  Then 无死锁状态

# Epic 2：识别质量

---

## Feature 2.1 准确率

### US-2.1.1 高质量识别

**User Story**

- As 用户  
- I want 获得准确识别结果  
- So that 可以信任设备输出  

**验收标准**

- Given 标准测试集  
  When 执行识别  
  Then Top-1准确率 ≥85%  

- Given 模型部署  
  When 系统运行  
  Then 支持 ≥30类植物  

---

## Feature 2.2 拒识机制

### US-2.2.1 低置信度处理

**User Story**

- As 用户  
- I want 系统避免错误识别  
- So that 不被误导  

**验收标准**

- Given 置信度 <60%  
  When 推理完成  
  Then 输出“未识别”  

- Given 低置信度  
  When 系统判断  
  Then 不输出错误类别

# Epic 3：区域采样系统

---

## Feature 3.1 模式入口

### US-3.1.1 模式选择

**User Story**

- As 用户  
- I want 选择系统模式  
- So that 进入不同功能场景  

**验收标准**

- Given 系统启动  
  When 进入首页  
  Then 显示模式选择界面  

- Given 首页  
  When 用户操作  
  Then 提供“识别模式”和“采样模式”  

- Given 用户操作  
  When 按键切换  
  Then 当前选项高亮  

---

## Feature 3.2 地图选择

### US-3.2.1 采样任务选择

**User Story**

- As 用户  
- I want 选择采样任务  
- So that 在特定场景下记录数据  

**验收标准**

- Given 进入采样模式  
  When 显示地图选择页  
  Then 至少显示3个地图  

- Given 地图选择页  
  When 页面渲染  
  Then 当前地图以主图或主卡片高亮显示  

- Given 地图选择页  
  When 页面渲染  
  Then 显示缩略图列表供导航切换  

- Given 地图选择页  
  When 用户确认当前地图  
  Then 可进入区域选择页  

- Given 地图选择页  
  When 用户执行长导航  
  Then 进入地图汇总统计页  

- Given 无地图  
  When 进入页面  
  Then 显示“无可用地图”  

---

## Feature 3.3 区域选择

### US-3.3.1 区域选择

**User Story**

- As 用户  
- I want 选择采样区域  
- So that 分类记录植物  

**验收标准**

- Given 进入地图  
  When 显示区域选择页  
  Then 至少显示4个区域  

- Given 区域选择页  
  When 页面渲染  
  Then 当前区域以主图或主卡片高亮显示  

- Given 区域选择页  
  When 页面渲染  
  Then 显示缩略图列表供导航切换  

- Given 选择区域  
  When 确认  
  Then 进入识别界面  

- Given 区域选择页  
  When 用户执行长导航  
  Then 进入区域统计页  

---

## Feature 3.4 区域识别

### US-3.4.1 区域内识别

**User Story**

- As 用户  
- I want 在区域内识别植物  
- So that 采集区域数据  

**验收标准**

- Given 已进入区域  
  When 用户操作  
  Then 执行识别流程  

---

## Feature 3.5 自动记录

### US-3.5.1 自动记录识别结果

**User Story**

- As 用户  
- I want 自动记录识别结果  
- So that 无需手动管理数据  

**验收标准**

- Given 识别成功  
  When 置信度 ≥60%  
  Then 写入记录  

- Given 识别失败  
  When 置信度 <60%  
  Then 不记录  

- Given 写入记录  
  When 数据生成  
  Then 包含名称  

- Given 云端返回植物中文名且本地标签目录中不存在对应 `plant_key`  
  When 写入记录  
  Then 系统生成稳定的命名空间 `plant_key` 并允许入库  

- Given 写入记录  
  When 数据生成  
  Then 包含置信度  

- Given 写入记录  
  When 数据生成  
  Then 包含时间戳  

- Given 存储失败  
  When 写入数据  
  Then 显示错误提示  

---

## Feature 3.6 去重与统计

### US-3.6.1 数据统计

**User Story**

- As 用户  
- I want 查看统计结果  
- So that 了解区域内植物情况  

**验收标准**

- Given 同一植物  
  When 多次识别  
  Then 不新增重复条目  

- Given 同一植物  
  When 再次识别  
  Then count +1  

---

## Feature 3.7 数据持久化

### US-3.7.1 数据保存

**User Story**

- As 用户  
- I want 数据长期保存  
- So that 可持续查看结果  

**验收标准**

- Given 写入数据  
  When 操作完成  
  Then 保存为本地JSON  

- Given 系统重启  
  When 重新进入  
  Then 数据仍存在  

- Given 数据读取  
  When 系统加载  
  Then 成功率100%  

- Given 数据损坏  
  When 读取失败  
  Then 提示错误  

---

## Feature 3.8 数据查看

### US-3.8.1 查看统计结果

**User Story**

- As 用户  
- I want 查看采样结果  
- So that 了解植物分布  

**验收标准**

- Given 进入区域数据页  
  When 查看数据  
  Then 显示植物种类数  

- Given 数据展示  
  When 渲染  
  Then 显示每种数量  

### US-3.8.2 查看地图汇总统计

**User Story**

- As 用户  
- I want 在地图层级查看所有区域的植物汇总结果  
- So that 快速了解整张地图的采样概况  

**验收标准**

- Given 用户位于地图选择页  
  When 执行长导航  
  Then 进入地图汇总统计页  

- Given 地图汇总统计页  
  When 渲染完成  
  Then 显示植物种类总数与聚合列表  

- Given 地图汇总统计页  
  When 用户翻页  
  Then 能循环查看全部统计页

# 二、系统级验收标准

---

## 普通模式

```

取景 → 拍照 → 推理 → 显示 → 返回

```

---

## 采样模式

```

首页 → 地图 → 区域 → 识别 → 自动记录 → 继续

```

---

## 全局约束

- Given 完整流程  
  When 走云端成功路径执行  
  Then 识别链路 ≤3秒  

- Given 完整流程  
  When 走云端失败、本地回退路径执行  
  Then 识别链路 ≤4.5秒  

- Given 连续操作  
  When 执行30次  
  Then 无崩溃  

- Given 数据记录  
  When 写入  
  Then 准确
