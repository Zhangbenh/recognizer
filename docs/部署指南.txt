Plant Recognizer 部署指南
============================

v1.1 回填说明：系统产品定义已调整为“默认中文、云端优先识别、本地回退、支持地图汇总统计”的植物识别与区域采样系统。本指南以下配置与部署说明按该口径执行。

1. 适用环境
-----------
- 开发/联调：Windows + keyboard + mock
- 实机运行：Raspberry Pi + GPIO + real


2. 前置条件
-----------
2.1 通用
- Python 3.11+（当前项目已在 3.14 开发环境验证）
- 已拉取本仓库
- 模型文件存在：models/plant_classifier_int8.tflite

2.2 树莓派额外依赖
- 摄像头库：python3-picamera2
- 屏幕渲染库：python3-pygame
- GPIO 库：python3-rpi.gpio
- 推理运行时：ai-edge-litert（优先）

推荐安装命令（Raspberry Pi）：

sudo apt update
sudo apt install -y python3-picamera2 python3-pygame python3-rpi.gpio
python3 -m pip install --upgrade pip
python3 -m pip install ai-edge-litert numpy pillow

注意：
- 若使用 venv，请确保运行时与安装依赖使用同一解释器。
- 若报 No TFLite runtime found，优先检查 sys.executable 对应环境是否安装了 ai-edge-litert。


3. 配置与模型路径
------------------
固定约定：
- labels: config/labels.json
- model manifest: config/model_manifest.json
- sampling config: config/sampling_config.json
- system config: config/system_config.json
- cloud config: config/cloud_config.json
- baidu mapping: config/baidu_plant_mapping.json
- model file: models/plant_classifier_int8.tflite
- data file: data/sampling_records.json
- assets: assets/fonts, assets/maps, assets/regions

补充说明：
- `system_config.json` 在 v1.1 下新增 `ui_language`、`recognition_strategy`、`cloud_request_timeout_s`、`local_infer_timeout_s`
- 运行时真实落盘数据统一为 `data/sampling_records.json`，内部结构按 `regions.<region_id>.records.<plant_key>` 组织

3.1 v1.1 新增配置准备
----------------------
- `config/cloud_config.json`：保留接口地址、超时、重试与凭据环境变量名；默认 token 缓存文件为 `data/.baidu_token_cache.json`
- `config/baidu_plant_mapping.json`：维护“百度中文名 -> 本地 plant_key”映射；未命中映射时，系统会生成稳定的 `cloud:` 命名空间键入统
- `config/sampling_config.json`：v1.1 交付时应同时校验地图/区域中文显示名与缩略图路径

3.2 云端环境变量覆盖优先级
--------------------------
敏感信息优先级固定为：
- `RECOGNIZER_BAIDU_API_KEY` / `RECOGNIZER_BAIDU_SECRET_KEY` 直接值
- `RECOGNIZER_BAIDU_API_KEY_ENV` / `RECOGNIZER_BAIDU_SECRET_KEY_ENV` 指向的环境变量名
- `config/cloud_config.json` 中的 `api_key_env` / `secret_key_env`

非敏感字段可按需使用以下覆盖项：
- `RECOGNIZER_BAIDU_API_ENDPOINT`
- `RECOGNIZER_BAIDU_TOKEN_ENDPOINT`
- `RECOGNIZER_BAIDU_REQUEST_TIMEOUT_S`
- `RECOGNIZER_BAIDU_RETRY_COUNT`
- `RECOGNIZER_BAIDU_TOKEN_CACHE_FILE`

部署建议：
- 版本库中的 `cloud_config.json` 只保留环境变量名，不直接写入明文凭据
- 如需让云端结果复用本地目录与统计名，请同步维护 `baidu_plant_mapping.json`


4. 启动方式
-----------
4.1 Windows（mock）

.venv\Scripts\python.exe app\main.py --runtime mock --input keyboard --log-level INFO

或：

powershell -ExecutionPolicy Bypass -File scripts/run_mock.ps1

4.2 Raspberry Pi（real）

SDL_VIDEODRIVER=kmsdrm python3 app/main.py --runtime real --input gpio --ui-backend screen --log-level INFO

或：

bash scripts/run_real_pi.sh

说明：
- 若只需要终端日志，可改为 `--ui-backend text`
- 若希望同时保留日志和屏幕画面，可使用 `--ui-backend both`
- 黑边优化默认已开启：`RECOGNIZER_SCREEN_FULLSCREEN=1` 与 `RECOGNIZER_SCREEN_FILL=1`
- 如需手动旋转屏幕预览，可设置 `RECOGNIZER_PREVIEW_ROTATION=90/180/270`
- 字体大小可通过 `RECOGNIZER_UI_SCALE` 调节（默认 1.25，建议范围 1.1~1.6）
- real 模式下若云端失败，系统会在 `INFERENCING` 内自动回退本地模型


5. 验收命令
-----------
5.1 全量测试

.venv\Scripts\python.exe -m pytest -q

5.2 Phase 4（性能）

python scripts/phase4_real_runtime_acceptance.py --runtime real --input keyboard --scenario all --samples 30 --warmup 5 --sleep-s 0.02 --output reports/phase4_real_runtime_report.json

5.3 Phase 5（采样）

python scripts/phase5_sampling_mode_acceptance.py --runtime real --input keyboard --scenario all --cycles 20 --map-nav 1 --region-nav 2 --output reports/phase5_sampling_mode_report.json

说明：
- `--scenario all` 会顺序执行 `cloud_success`、`cloud_fallback`、`local_only` 三条确定性链路，并把结果汇总到 `scenario_results`
- 如需单独排查某一路径，可改为 `--scenario cloud_success`、`--scenario cloud_fallback` 或 `--scenario local_only`
- 上述脚本中的 `cloud_success` / `cloud_fallback` 为确定性验收场景，不等价于真实百度云连通性证明；真实网络与凭据联调应通过主程序 real 模式手测补充
- Phase 5 自动化主要验证状态流、录制计数与统计页数据；screen 后端图片页与 `MAP_STATS` 的可视确认建议补一轮手测

5.4 Phase 6（收口检查）

powershell -ExecutionPolicy Bypass -File scripts/phase6_release_check.ps1

5.5 Phase 7 实机签收（第三阶段）

方式 A（逐条）：

 python3 scripts/phase4_real_runtime_acceptance.py --runtime real --input keyboard --scenario all --samples 30 --warmup 5 --sleep-s 0.02 --output reports/phase7_phase4_real_report.json
 python3 scripts/phase5_sampling_mode_acceptance.py --runtime real --input keyboard --scenario all --cycles 20 --map-nav 1 --region-nav 2 --output reports/phase7_phase5_real_report.json

方式 B（一键）：

bash scripts/phase7_real_signoff_pi.sh

通过判据：
- Phase4：reports/phase7_phase4_real_report.json 的 `pass.cloud_success=true`、`pass.cloud_fallback=true`、`pass.local_only=true`、`pass.overall=true`
- Phase5：reports/phase7_phase5_real_report.json 的 `pass.cloud_success=true`、`pass.cloud_fallback=true`、`pass.local_only=true`、`pass.overall=true`
- 如需定位具体失败链路，查看对应报告的 `scenario_results.<scenario>`
- 总签收：Phase4 overall=true 且 Phase5 overall=true
- 真实百度云连通性、screen 图片页与 `MAP_STATS` 导航需额外手测一次
- 详细清单见：docs/phase7_实机签收清单.txt


6. 故障排查
-----------
6.1 No TFLite runtime found
- 原因：当前解释器环境没有推理运行时
- 处理：使用同一解释器安装 ai-edge-litert

6.2 input adapter does not support simulated events
- 原因：phase5 验收脚本使用了 gpio 输入
- 处理：改用 --input keyboard

6.3 摄像头启动失败
- 检查 CSI 排线、权限与 picamera2 安装
- 先用 real 模式最小命令验证启动

6.4 GPIO 无响应
- 检查 BCM 编号（BTN1=17, BTN2=18）
- 检查 pull-up 与按键到 GND 接线


7. 交付建议
-----------
- 交付时附带 reports/ 下最新 phase4/phase5 报告
- Phase 7 实机签收时附带 phase7_phase4_real_report.json 与 phase7_phase5_real_report.json
- 交付包同时附带 docs/v1.1_升级说明.txt，明确新增配置文件、环境变量与回滚注意事项
- 记录目标设备型号、系统版本、Python 版本
- 不要把百度 API 明文凭据写入仓库文件；交付时仅交接环境变量配置方式
- 升级前备份 data/sampling_records.json 与本地配置覆盖项
- 保留 docs/已知问题清单.txt 便于后续迭代
