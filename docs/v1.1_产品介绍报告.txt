Recognizer v1.1 产品介绍报告
=============================

文档信息
--------
- 文档版本：v1.1
- 文档日期：2026-04-24
- 适用对象：项目负责人、合作方、老师/评审、潜在使用部门、演示观众
- 文档定位：用于快速介绍当前版本系统的产品价值、核心能力、运行形态与成熟度


一、一句话介绍
----------------
Recognizer v1.1 是一套面向树莓派小屏终端的便携式植物识别与区域采样系统，支持云端优先识别、本地模型自动回退、中文交互界面、区域采样记录以及地图级汇总统计。


二、产品定位
-------------
本产品的目标不是单纯做一个“拍照出结果”的识别器，而是提供一套可以在实际采样流程中使用的轻量终端系统，帮助用户完成以下任务：

1. 在现场快速识别当前目标是否为已知植物。
2. 按地图和区域组织采样任务，减少记录混乱。
3. 在网络正常时优先利用云端识别提升成功率。
4. 在云端不可用时自动回退本地模型，保证流程不中断。
5. 通过区域统计和地图统计，快速查看当前采样覆盖情况。

它适合用于教学演示、实验室采样、小规模场地巡检、农业或园艺场景的轻量信息采集。


三、当前版本核心能力
---------------------
1. 普通识别模式
- 用户在预览页拍摄后，系统执行识别并展示名称、置信度和识别来源。

2. 采样记录模式
- 用户先选择地图，再选择区域，随后进入连续“预览 -> 拍摄 -> 识别 -> 自动记录”的工作流。

3. 云端优先，本地回退
- 当前版本默认识别策略为 cloud_first。
- 云端优先调用百度植物识别接口。
- 当云端失败、超时、无结果或不满足阈值条件时，自动回退本地模型。
- 识别来源会在界面中明确显示，便于用户判断结果来自云端还是本地回退。

4. 中文界面与图片化选择
- 系统默认中文显示。
- 地图页和区域页采用图片化选择方式，更适合小屏设备与非技术用户。

5. 统计能力
- 支持区域级采样统计。
- 支持地图级聚合统计，便于从更高层级查看整体采样结果。

6. 设备端运行
- 当前实机目标平台为 Raspberry Pi。
- 支持 480x800 竖屏小屏显示。
- 支持双按键输入语义，适合无触摸或低成本交互终端。


四、当前版本规模与配置事实
-----------------------------
根据当前仓库配置，v1.1 版本具备以下资源规模：

- 地图数量：4 张
- 区域数量：24 个
- 本地标签类别：30 类
- 默认识别策略：cloud_first
- 本地识别阈值：0.6
- 云端识别阈值：0.5
- 默认界面语言：中文

这意味着当前版本已经不是原始的单场景识别原型，而是一套具备多地图、多区域、多植物类别组织能力的系统版本。


五、典型用户流程
-----------------
1. 普通模式

首页 -> 预览 -> 拍摄 -> 识别 -> 结果页 -> 返回预览

适用于快速识别当前目标，不产生采样记录。

2. 采样模式

首页 -> 地图选择 -> 区域选择 -> 预览 -> 拍摄 -> 识别 -> 自动记录 -> 返回预览

适用于需要按地点持续记录的任务场景。

3. 统计查看

地图选择 -> 地图统计

或

区域选择 -> 区域统计

适用于查看阶段性采样结果和覆盖情况。


六、产品亮点
-------------
1. 产品能力完整
- 不是单点算法演示，而是“识别 + 采样 + 统计 + 设备交互”闭环系统。

2. 实际可用性更强
- 云端优先提高识别覆盖率。
- 本地回退保证在网络异常时仍能继续工作。

3. 交互贴近终端场景
- 中文界面降低使用门槛。
- 小屏与双键交互降低硬件复杂度。
- 地图/区域图片化选择更利于现场操作。

4. 架构清晰、可维护
- 单线程事件驱动状态机作为唯一控制流入口。
- 配置、领域、控制器、基础设施与呈现层边界清晰。
- 既便于后续继续迭代，也便于教学展示系统设计方法。


七、技术实现概览
-----------------
1. 控制核心
- 系统采用单线程事件驱动状态机。
- 所有页面切换、识别动作、记录动作和错误处理都通过统一状态流完成。

2. 识别链路
- 拍摄后先尝试云端识别。
- 云端成功则优先返回云端结果。
- 云端失败或不满足条件时自动走本地模型。
- 最终统一输出 RecognitionResult，保证页面展示和统计入库逻辑一致。

3. 数据管理
- 采样统计当前存储于本地 JSON 文件 data/sampling_records.json。
- 统计主键统一为 plant_key，便于兼容本地标签与云端扩展键。

4. 设备端显示
- screen 后端负责小屏渲染。
- text 后端继续保留，用于测试、联调与回归验证。

九、适合怎样介绍给别人
-------------------------
如果面对非技术听众，可以这样介绍：

“这是一个运行在树莓派小屏设备上的植物识别与采样终端。它不只是识别植物，还能按地图和区域完成现场采样记录，并把结果自动汇总成统计信息。系统默认优先使用云端识别以提高成功率，但在网络不稳定时又能自动切换到本地模型，保证流程不断。”

如果面对偏技术或评审对象，可以强调：

“这个版本的重点不是单模型精度，而是完成了终端交互、识别编排、数据入库和统计展示的完整闭环。系统以单线程状态机为核心，支持云端优先、本地回退的双链路识别策略，并已经通过自动化回归和树莓派实机签收。”


十、建议演示顺序
-----------------
建议向别人介绍时，按以下顺序演示：

1. 首页与双模式切换
- 说明系统有普通识别和采样记录两种工作方式。

2. 地图和区域选择
- 展示图片化选择页，说明系统支持按场地组织数据。

3. 现场识别
- 拍摄并展示结果页，说明来源、置信度与回退逻辑。

4. 自动记录
- 在采样模式下完成一次识别，展示自动计入统计。

5. 统计查看
- 展示区域统计或地图统计，说明系统最终交付的是结构化采样结果，而不是单次识别截图。


十一、当前版本边界
-------------------
为了保证介绍口径准确，建议同时说明以下边界：

1. real 硬件链路当前面向 Raspberry Pi，不面向 Windows 实机运行。
2. 统计数据当前使用 JSON 存储，适合轻量场景，不是数据库方案。
3. 云端识别依赖网络和凭据配置，现场演示前应提前完成连通性检查。
4. 当前版本的目标是“可演示、可采样、可统计、可扩展”，不是大规模商用量产版。


十二、结论
-----------
Recognizer v1.1 已从“离线识别原型”发展为“云端优先、本地回退、支持地图区域采样与统计的小型终端系统”。

它的价值不只是识别一次植物，而是把识别能力放进一个完整、可运行、可展示、可扩展的采样工作流中。对于课程设计、项目展示、实机演示和小规模试点，这个版本已经具备较强的说明力和展示价值。
