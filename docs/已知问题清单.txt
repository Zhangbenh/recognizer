已知问题清单（Phase 6）
========================

1. 环境与平台限制
-----------------
1) Windows 不支持 real 运行链路
- 原因：picamera2 与 RPi.GPIO 面向树莓派
- 影响：Windows 仅建议使用 mock + keyboard

2) GPIO 场景无法直接用于 phase5 自动化脚本
- 原因：脚本依赖模拟按键注入，GPIO 适配器不支持 push_simulated_event
- 影响：GPIO 建议手工验收

3) Python 版本与推理运行时兼容性
- 原因：不同平台 wheel 可用性差异
- 建议：树莓派优先 ai-edge-litert；确保安装与运行解释器一致


2. 功能与行为约束
------------------
1) RECORDING 状态可能是瞬态
- 现象：单次 tick 可能完成 DISPLAY->RECORDING->RECORD_OK->PREVIEW
- 说明：验收不要用“是否看到 RECORDING 页面”做唯一依据
- 建议：使用统计增量（recorded_delta_count）判定

2) STATS 异常采用降级策略
- 现象：数据损坏时停留 STATS 并显示 warning
- 说明：符合“提示 + 安全流程继续”设计

3) 非致命错误提示为短时提示
- 现象：PREVIEW 中仅显示一轮提示，随后自动清理
- 说明：避免旧错误长期污染后续正常流程


3. 数据与运维风险
-----------------
1) data/sampling_records.json 体积增长
- 风险：长期采样后文件增大、读写延迟上升
- 建议：定期归档并重置历史数据

2) 配置文件手工修改风险
- 风险：labels/model_manifest/sampling_config 格式错误会触发启动错误
- 建议：修改后先运行 pytest 与 mock 启动验证


4. 后续优化方向
---------------
1) 增加实机长稳压测脚本（>=1h）并自动导出内存/CPU趋势
2) 增加数据文件轮转策略与压缩归档工具
3) 增加 ERROR 页面更细粒度修复建议（按错误类型给出操作指引）
4) 增加部署前检查工具（配置校验、模型校验、依赖校验）
