Phase 4 硬件侧附加验收指南（Real Runtime）
=================================================

v1.1 回填说明
------------
本指南保留为“硬件侧实时链路基线验收”说明。v1.1 新增云端优先识别后，整体时延口径拆分为 `cloud_success`、`cloud_fallback`、`local_only` 三条链路；若执行 v1.1 最终验收，建议以 `--scenario all` 输出分链路报告，并补一轮真实网络连通性手测，不应仅以本指南作为唯一云端验收依据。

1. 目的
-------
本指南用于在树莓派上执行 Phase 4 的硬件侧附加验收：
- 相机采集时延采样（capture）
- 真实模型推理时延采样（infer）
- 单次识别全流程时延采样（full_flow）
- 按 config/system_config.json 的 performance_budget 做门槛判定


2. 脚本位置
-----------
scripts/phase4_real_runtime_acceptance.py


3. 前置条件
-----------
1) 已在树莓派上能运行主程序 real 模式（命令与当前目录有关）：
   - 若当前在仓库根目录 recognizer：
     python app/main.py --runtime real --input gpio --log-level INFO
   - 若当前在 app 子目录：
     python main.py --runtime real --input gpio --log-level INFO

2) 依赖就绪：
- 摄像头：picamera2
- 推理运行时：ai-edge-litert / tflite-runtime / tensorflow 任一可用

3) 推荐当前工作目录为仓库根目录 recognizer
  - 可先执行 pwd 确认当前目录
  - 若你当前在 app 子目录，建议先执行 cd .. 回到仓库根目录


4. 推荐执行命令
---------------
以下命令默认在仓库根目录 recognizer 执行。

4.1 标准验收（建议）
python scripts/phase4_real_runtime_acceptance.py \
  --runtime real \
  --input keyboard \
  --scenario all \
  --samples 30 \
  --warmup 5 \
  --sleep-s 0.02 \
  --log-level INFO \
  --output reports/phase4_real_runtime_report.json

说明：
- input 选 keyboard 即可，本脚本不依赖按键输入轮询。
- 若你希望和主程序保持一致，也可改为 --input gpio。
- `--scenario all` 会把三条确定性链路的结果汇总到顶层报告，并在 `scenario_results` 保留分场景明细。
- 该脚本的 `cloud_success` / `cloud_fallback` 用于稳定复验编排与预算，不替代真实百度云连通性证明。


4.2 打印每次样本（排障用）
python scripts/phase4_real_runtime_acceptance.py \
  --runtime real \
  --input keyboard \
  --scenario cloud_fallback \
  --samples 20 \
  --warmup 3 \
  --print-samples \
  --output reports/phase4_real_runtime_report_verbose.json

若你当前在 app 子目录，可改用：
python ../scripts/phase4_real_runtime_acceptance.py ...


4.3 链路隔离复验（可选）
python scripts/phase4_real_runtime_acceptance.py --runtime real --input keyboard --scenario cloud_success --samples 20 --warmup 3 --output reports/phase4_cloud_success.json
python scripts/phase4_real_runtime_acceptance.py --runtime real --input keyboard --scenario cloud_fallback --samples 20 --warmup 3 --output reports/phase4_cloud_fallback.json
python scripts/phase4_real_runtime_acceptance.py --runtime real --input keyboard --scenario local_only --samples 20 --warmup 3 --output reports/phase4_local_only.json


5. 判定规则
-----------
脚本当前以 p95 门槛判定。

当使用单场景模式时，通过项如下：
- capture_p95 <= 当前场景 budget.capture_s
- infer_p95 <= 当前场景 budget.infer_s
- full_flow_p95 <= 当前场景 budget.full_flow_s
- expected_path = true

当使用 `--scenario all` 时，建议同时满足：
- pass.cloud_success = true
- pass.cloud_fallback = true
- pass.local_only = true
- pass.overall = true
- scenario_results.<scenario>.pass.expected_path = true


6. 报告输出
-----------
脚本会输出 JSON 报告，包含：
- 采样参数（samples/warmup/sleep）
- 设备与 Python 信息
- 顶层 `scenario` 与分链路 `scenario_results`
- `budget_s.paths` 中三条链路的预算口径
- capture/infer/full_flow 的统计值（min/max/mean/median/p95/p99/stdev）
- 识别比例与平均置信度
- pass 结果（分场景 + overall）

说明：
- 顶层 `metrics` 是所有已执行场景的汇总统计；排障与定位应优先查看 `scenario_results.<scenario>.metrics`
- 若要确认真实百度云可用性，请另外在主程序 real 模式下完成联网手测


7. 快速复验建议
---------------
建议至少执行两轮：
1) 空闲状态一轮（系统冷启动后）
2) 连续运行后一轮（系统热态）

若两轮都 overall=True，可作为硬件侧附加验收通过依据。


8. 常见问题
-----------
Q1: 报 No TFLite runtime found
A1: 当前解释器未安装推理运行时。优先安装 ai-edge-litert，并确认脚本使用的 Python 与安装目标一致。

Q2: 摄像头启动失败
A2: 检查摄像头连接、权限与 picamera2 环境；先用主程序 real 模式验证摄像头链路。

Q3: full_flow 超预算但 infer 正常
A3: 优先检查相机采集耗时、系统负载、I/O 抖动；可适当加 --sleep-s 观察抖动变化。

Q4: `--scenario all` 通过了，是否等于真实云端可用？
A4: 不等于。该脚本的多场景模式用于稳定验证识别编排与预算口径；真实百度云可用性仍需在配置好凭据和网络后，通过主程序 real 模式额外手测。
